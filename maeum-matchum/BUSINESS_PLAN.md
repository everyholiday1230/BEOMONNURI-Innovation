# 마음맞춤 (MaeumMatchum) 사업계획서

> 연애의 모든 것 — AI가 연애와 결혼의 의사결정을 도와주는 Agent
> AI 판단 → 근거 제시 → 추천 → 보완 제시 → 실행 지원

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | 마음맞춤 (MaeumMatchum) |
| 한 줄 정의 | 10대 핵심 영역 진단 기반 연애·결혼 의사결정 AI 에이전트 |
| 타깃 | 진지한 관계·결혼을 고민하는 25~40세, 소개팅/연애 초기 커플 |
| 핵심 가치 | "감정만이 아닌 데이터와 AI로, 후회 없는 관계 결정" |
| AI 모델 | HyperCLOVA X SEED 0.5B (경량·상업적 사용 가능·셀프호스팅으로 저비용) |

### 왜 SEED 0.5B 인가
- **초저비용**: 0.5B 경량 모델이라 GPU 1장(또는 CPU)으로도 셀프호스팅 가능 → API 종량과금 대비 사용자당 추론비 획기적 절감.
- **상업적 활용 가능**: 오픈 가중치 모델로 서비스에 내장 가능.
- **한국어 특화**: 네이버 HyperCLOVA X 계열로 한국어 연애 상담 뉘앙스에 강점.
- **안전한 폴백 설계**: 모델이 없거나 실패해도 **결정론적 규칙 엔진**이 완결된 리포트를 생성 → 서비스 안정성 확보.

## 2. 핵심 기능 (에이전트 5단계 플로우)

사용자가 나와 상대를 각각 진단하면, 에이전트가 다음 5단계로 의사결정을 돕는다.

1. **AI 판단 (Verdict)** — 종합 궁합 점수(0~100)와 5등급 판단
   (`천생연분 / 좋은 궁합 / 노력하면 충분 / 신중한 접근 필요 / 차이가 큰 관계`)
2. **근거 제시 (Evidence)** — 강점 영역·차이 영역을 점수와 함께 구체적으로 제시
3. **추천 (Recommend)** — 관계 진전 여부에 대한 방향 제안
4. **보완 제시 (Improve)** — 취약 영역별 개선 포인트
5. **실행 지원 (Action)** — 오늘 당장 해볼 수 있는 구체적 액션 아이템 + 1:1 AI 상담 대화

### 10대 진단 영역 (가중치)
| 영역 | 가중치 | 매칭 방식 | 설명 |
|------|:----:|------|------|
| 성향 | 1.0 | 유사도 | 내향/외향, 계획/즉흥 |
| 가치관 | 1.4 | 유사도 | 삶의 우선순위 |
| 생활패턴 | 1.0 | 유사도 | 생활 리듬·정리 성향 |
| 소비습관 | 1.3 | 유사도 | 저축/소비 성향 (부부 갈등 1순위) |
| 미래계획 | 1.4 | 유사도 | 결혼·자녀·거주 |
| 가족관 | 1.2 | 유사도 | 원가족·명절·부양 |
| 직업 안정성 | 1.1 | 유사도 | 커리어 리스크 수용도 |
| 대화 스타일 | 1.2 | 유사도 | 감정 표현·연락 빈도 |
| 애착유형 | 1.5 | 조합표 | 안정형/불안형/회피형 (관계 안정성 핵심) |
| 갈등해결방식 | 1.3 | 유사도 | 다툼 대처 방식 |

> 애착유형은 단순 유사도가 아닌 **조합 궁합표**(예: 안정형×안정형=95, 불안형×회피형=35)로 계산해 심리학적 근거를 반영.

## 3. 데이터베이스 설계 (SQLite)

경량 서비스에 맞춰 SQLite로 시작하고, 트래픽 증가 시 PostgreSQL로 이관 가능한 구조.

```
users (사용자)
 ├─ id, nickname, age, gender, created_at

profiles (진단 프로필: 본인/상대)
 ├─ id, user_id(FK), owner_type('self'|'partner'), label
 ├─ answers_json  ── 원본 응답
 ├─ scores_json   ── 차원별 0~100 점수
 ├─ attachment    ── secure|anxious|avoidant
 └─ created_at

analyses (궁합 분석 결과)
 ├─ id, user_id(FK), self_profile_id(FK), partner_profile_id(FK)
 ├─ overall_score, verdict
 ├─ dimension_json ── 영역별 상세
 ├─ report_json    ── 판단>근거>추천>보완>실행 리포트
 ├─ ai_used        ── AI 사용 여부
 └─ created_at

messages (에이전트 대화 기록)
 ├─ id, analysis_id(FK), role('user'|'assistant'), content, created_at
```

- 정규화된 관계형 설계 + JSON 컬럼으로 유연성 확보.
- `ON DELETE CASCADE`로 사용자 삭제 시 관련 데이터 자동 정리(개인정보 보호).
- WAL 모드로 동시 읽기 성능 확보.

## 4. 시스템 아키텍처

```
[반응형 웹 (Vanilla JS SPA)]
        │ REST/JSON
        ▼
[Node.js + Express API]
   ├─ scoring.js  : 응답→점수, 궁합 계산 (결정론적)
   ├─ agent.js    : 5단계 리포트 생성 + 대화
   └─ aiClient.js : OpenAI 호환 호출 (SEED 0.5B)
        │                         │ (실패/부재 시)
        ▼                         ▼
[SQLite DB]              [규칙 엔진 폴백]
        ▲
        │ OpenAI 호환 API
[HyperCLOVA X SEED 0.5B]
  (vLLM / Ollama / sglang 셀프호스팅)
```

**설계 원칙**: AI는 "문장을 다듬고 대화하는" 역할, 점수·판단의 뼈대는 **결정론적 엔진**이 담당.
→ 재현성·설명가능성 확보 + AI 장애에도 서비스 지속.

## 6. 시장 분석

- **시장 규모**: 국내 데이팅앱 시장 연 수천억 원 규모, 결혼정보 서비스 시장 성장세. "결정 피로"를 겪는 진지한 관계 이용자층 확대.
- **문제점**: 기존 데이팅앱은 "매칭"에 집중, **관계를 이어갈지/결혼할지의 의사결정**은 개인 몫으로 방치됨.
- **차별점**:
  1. 매칭이 아닌 **의사결정 지원**에 특화 (블루오션 포지셔닝)
  2. 감정 상담을 넘어 **데이터 기반 5단계 구조화된 조언**
  3. 한국어 특화 경량 AI로 **저비용·빠른 응답**
- **경쟁 우위**: 심리학 기반 애착유형 매칭 + 설명 가능한 점수 체계 + AI 대화의 결합.

## 8. 리스크 및 대응
- **AI 조언의 책임 한계** → "참고용, 최종 결정은 본인" 명시 + 극단적 상황(이별·위기) 시 전문가 안내.
- **개인정보 보호** → 최소 수집, 회원가입 없는 진단, CASCADE 삭제, 향후 암호화·동의 관리 도입.
- **AI 품질 편차** → 결정론적 엔진이 하한선 보장, AI는 개선 레이어로만 활용.
